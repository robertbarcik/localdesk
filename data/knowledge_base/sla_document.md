## 2. Priority Definitions

- **Critical (P1):** Complete service outage or security breach affecting multiple users or business-critical systems. Examples: email server down, network outage, ransomware detection, ERP system unavailable.
- **High (P2):** Significant degradation of service affecting a team or department. Examples: shared printer offline for a floor, VPN instability for remote team, database performance severely degraded.
- **Medium (P3):** Individual user impacted but workaround available. Examples: single user VPN issue, software crash with workaround, peripheral device malfunction.
- **Low (P4):** Minor inconvenience or informational request. Examples: how-to questions, feature requests, cosmetic issues, scheduled maintenance inquiries.

---

## 3. Escalation Paths

### Standard Escalation
1. **Level 1 (L1):** Service desk agent — initial triage, known-issue resolution, KB-guided troubleshooting.
2. **Level 2 (L2):** Technical specialist — hands-on diagnosis, remote access troubleshooting, configuration changes.
3. **Level 3 (L3):** Engineering team — root cause analysis, code/infrastructure fixes, vendor coordination.

### Management Escalation
- Triggered when resolution exceeds escalation threshold for the customer's tier.
- Gold: escalated to IT Service Manager and account manager simultaneously.
- Silver/Bronze: escalated to IT Service Manager.

---

## 4. Reporting and Reviews

- **Gold:** Weekly incident summary report, monthly SLA compliance review meeting.
- **Silver:** Monthly incident summary report, quarterly SLA compliance review.
- **Bronze:** Quarterly summary report only.

---

## 5. Exclusions

SLA commitments do not apply to:
- Force majeure events (natural disasters, war, pandemic restrictions)
- Issues caused by unauthorized changes made by the Client
- Third-party service outages outside Provider's control (e.g., ISP failures, cloud provider outages)
- Scheduled maintenance windows (communicated 5 business days in advance)
